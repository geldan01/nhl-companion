# NHL App Shell & Navigation — Implementation Plan

**Spec:** [`docs/superpowers/specs/2026-05-09-app-shell-and-navigation-design.md`](../specs/2026-05-09-app-shell-and-navigation-design.md)
**Status:** Not started.

Phases run top-to-bottom. Each step lists files touched, acceptance criteria, and any verification commands. Tick the checkbox when the step is done **and** its acceptance criteria pass.

The plan is structured so an interruption between phases leaves the codebase in a working state (`npm run lint`, `npm run build`, `npm run test:run` all green). Phase 2 is the first to introduce Playwright; from Phase 2 onward, `npm run test:e2e` must also pass.

Each phase produces a short stack of small commits (one feature per commit) and ends with a phase-closing commit that's the natural rebase point.

## Phase 0 — Foundation

Everything every later phase depends on: image hosts, design tokens, the team-color map, and the shared component primitives. No routes change yet; `app/page.tsx` keeps its boilerplate until Phase 1.

- [x] **0.1 Allow-list NHL CDN hosts in `next.config.ts`.** Done. Single host (`assets.nhle.com`); two patterns: `/logos/**` and `/mugs/**`. Two non-obvious deviations from the step description: (a) team logos are SVG, which Next blocks by default, so set `dangerouslyAllowSVG: true` + a script-blocking CSP (`default-src 'self'; script-src 'none'; sandbox;`); (b) used the **object form** (not the `new URL(...)` shorthand) and **omitted `search`** so logo URLs with `?season=YYYYYYYY` are accepted — the shorthand pins `search: ''` and rejects all queries. Verified by curling `/_next/image?url=...` against a running `npm run dev`: SVG logo (with and without query string), PNG mug, and JPG actionshot all → 200; an unallowed host and an unallowed path under the same host both → 400. No throwaway page needed.

- [x] **0.2 Define color tokens in `src/app/globals.css`.** Done. Added all 11 spec tokens under `:root` (light) with overrides under `@media (prefers-color-scheme: dark)`. Deferred the `.dark` class hook from the spec — there's no UI toggle to flip a class yet, and adding both branches now is dead code. Will revisit when a theme toggle lands. Kept the existing `--background`/`--foreground` aliases (now `var(--bg)` / `var(--text)`) so the boilerplate `page.tsx` keeps rendering until Phase 1.1 replaces it. `body` switched from `--background`/`--foreground` to `--bg`/`--text` directly. Visual smoke check deferred (no browser session); the existing page uses explicit Tailwind classes (`bg-zinc-50`, `dark:bg-black`, etc.) that don't read these tokens, so the body color is mostly hidden behind the page's own styling — render is unchanged. `npm run build` clean.

- [x] **0.3 `src/lib/team-colors.ts`.** Done. 32 teams (verified against `standings/__fixtures__/standings.json` — all `teamAbbrev.default` values present in the map and vice versa). Hex values track each team's official primary/secondary identity at the time of writing. Exports: `TeamCode` (string-literal union of the 32 codes), `TeamColors`, `TEAM_COLORS`, `FALLBACK_TEAM_COLORS` (neutral grey), `getTeamColors(code)` runtime fallback helper. Tests (`team-colors.test.ts`, 4 cases): map covers exactly the fixture's 32 codes; every primary and secondary matches `^#[0-9A-Fa-f]{6}$`; `getTeamColors` returns mapped colors for known codes and the fallback for unknown.

- [x] **0.4 `<DataState>` in `src/components/data-state.tsx` + test.** Done. Props: `isLoading`, `error`, `hasData`, `skeleton`, `emptyState?`, `children`. Branch order (matters): hard error first (4xx → empty state slot or default "Not found." / "Unavailable (5xx)."; schema or http ≥ 500 → top banner + stale children if any), then loading-skeleton (only when `!hasData`), then soft error (`network`/`timeout` → absolute-positioned "Delayed" badge over children), then plain children. Default empty state and banner internal — components consuming `<DataState>` only need to pass a custom `emptyState` when the default text is wrong. Tests (9 cases) cover every branch, including the precedence rule and the default-vs-custom empty state.

- [x] **0.5 `<Skeleton>` in `src/components/skeleton.tsx`.** Done. Four variants — `card` (h-28 rounded), `row` (h-8 rounded), `rink` (aspect-100/85), `pill` (inline-block h-5 w-24 rounded-full). Optional `count` repeats with `flex flex-col gap-2`. Optional `className` for layout overrides. `aria-hidden` on every block so screen readers skip the visual placeholder. No tests yet — pure-presentational, will be exercised by the integration tests in 1.10.

- [x] **0.6 `<TeamLogo>` in `src/components/team-logo.tsx`.** Done. URL convention: `https://assets.nhle.com/logos/nhl/svg/{code}_dark.svg` (the `_dark` SVG variant works on both light and dark UI backgrounds — outlines stay opaque). Wraps `next/image` with `unoptimized` because SVG optimization is a no-op (it's already vector); this also bypasses the `/_next/image` proxy entirely, so SVG logos don't actually rely on the `dangerouslyAllowSVG` flag from 0.1 — that flag only matters as a defense-in-depth for any future code that forgets to set `unoptimized`. Container is an inline-flex span with `background: primary` from `getTeamColors`, sized via the `size` prop (default 32) — this also doubles as the loading-state placeholder. `alt` is the team code.

- [x] **0.7 `<TeamChip>` in `src/components/team-chip.tsx`.** Done. `<TeamLogo>` (default 24px) + abbreviation (font-medium) + optional record in muted xs. Inline-flex with `gap-2`. Plain composition over `<TeamLogo>` — no extra logic.

- [x] **0.8 `<PlayerChip>` in `src/components/player-chip.tsx`.** Done — with a deviation. The plan called for props `{id, name}` and constructing the URL from id, but NHL headshots are season- and team-keyed (`/mugs/nhl/{season}/{team}/{id}.png`) — id alone is insufficient. Switched to props `{name, headshotUrl?}`: consumers thread the URL from the data layer (boxscore.player.headshot, player.headshot, etc.) and we render initials on a neutral circle when no URL is provided. `id` is dropped — callers use the parent component's `<Link href="/player/[id]">` for navigation.

- [x] **0.9 `<GameStatePill>` in `src/components/game-state-pill.tsx` + test.** Done. Props as specced. Pure formatting helper `pillText()` exported alongside the component for direct testability (no need to query rendered DOM for string assertions). LIVE/CRIT → `LIVE NP MM:SS`; FINAL/OFF → `FINAL`; PRE/FUT → locale-formatted time from `startTimeUTC` or "SCHEDULED" if missing; unknown states fall through to the raw value. Period beyond 3 → `OT`, `2OT`, `3OT`, etc. Live color = `var(--live)`, everything else = `var(--text-muted)`. 10 tests cover each state, OT periods, missing fields, and the color-class mapping. Type `KnownGameState` exported for callers that want narrowed input.

- [x] **0.10 Update `src/lib/nhl/index.ts` if needed.** Confirmed no-op. Components import `NhlApiError` via the existing `import type { NhlApiError } from "@/lib/nhl"` re-export; nothing else from this phase belongs in the data-layer surface.

The headline screen plus the persistent pill behavior. This is also where Playwright lands.

Skipped `--with-deps` (Linux apt-only flag; macOS doesn't need it). `npx playwright install chromium` pulled the Chromium browser + headless-shell binaries to `~/Library/Caches/ms-playwright/` (~258 MiB). Added `test:e2e` script to `package.json`. Vitest's `include` is already scoped to `src/**/*.{test,spec}.{ts,tsx}` so root-level `e2e/*.spec.ts` files won't get picked up by `vitest run` once they exist. Updated `.gitignore` for Playwright outputs (`test-results/`, `playwright-report/`, `playwright/.cache/`, `blob-report/`).

- [x] **2.2 `playwright.config.ts` + fixtures-server.** Done. Config boots `next dev -p 3100` with `NHL_FIXTURES_MODE=1`. `reuseExistingServer: !CI` so local re-runs are fast. Chromium project only. `webServer.timeout` 120s for cold cache. Tests live in `./e2e`. The fixture short-circuit lives in `src/lib/nhl/fixtures-mode.ts` (URL pattern → fixture map for all 12 endpoints) consumed by `nhlFetch` at the top of the function — single point of change, no per-route-handler edits, and the fixtures are tree-shaken from production bundles because `process.env.NHL_FIXTURES_MODE` is never `'1'` outside e2e. The fixture is still parsed by the schema so a regression in the recorded JSON surfaces as a `kind: 'schema'` error in the e2e itself. Verified locally: `NHL_FIXTURES_MODE=1 next dev -p 3100`, then `curl /api/nhl/standings` returned COL as the first team (matches the standings fixture); `curl /api/nhl/schedule-now` returned 2026-05-09 with 2 games (matches the scheduleNow fixture).

- [x] **2.3 `/game/[id]` route scaffold.** Done. `page.tsx` is a server component that awaits `params` (Next 16 convention) and renders `<GameDetail id={number} />`. Invalid (non-numeric) ids short-circuit to a friendly message before any client hook runs. `GameDetail` is the `'use client'` component. It consumes `useGame(id)` (canonical liveness hook) and wraps in `<DataState>` with the canonical empty-state ("Game not found.") for 4xx, the bigger skeleton (score-header card + 3-column rink/rows/rows) for loading, and a placeholder for the body until 2.4–2.7 land. `usePlayByPlay`/`useBoxscore` are deliberately deferred to the panes themselves so they each render independent skeletons. Build registers `ƒ /game/[id]` correctly.

- [x] **2.4 `<ScoreHeader>` in `src/components/game/score-header.tsx`.** Done. Sticky just below the app header (`top-12 z-20`). Three columns: away (logo + city + name + score), centered status (`<GameStatePill>` from period/clock; "Intermission" line when `clock.inIntermission`), home (mirrored — `flex-row-reverse`). `ml-auto` on the score in each column pushes both scores toward the center, so the layout reads `[TOR Logo Name 2 ... LIVE 2P 8:14 ... 1 Name Logo BOS]`. The `×` button on the right calls `clearWatching()` then `router.push('/')` so a deliberate "stop watching" doesn't get re-seeded by the page's mount effect on the next render (as captured in the spec). Wired into `<GameDetail>` — Skeleton state still renders the larger placeholder; happy path renders header + the body placeholder for 2.5–2.7.

- [x] **2.5 Tabs/multi-pane layout container.** Done. `src/lib/use-match-media.ts` is the SSR-safe matchMedia hook (returns `false` during SSR/hydration; updates after mount). `src/components/game/game-body.tsx` reads `useMatchMedia('(min-width: 1024px)')` and renders one of two layouts:
  - **mobile**: sticky tab bar (`Plays` / `Box` / `Shots`), tab state via URL hash. Default tab is `plays` and the hash is dropped for that case so the canonical URL stays clean.
  - **desktop**: 3-column grid (`grid-cols-[1.2fr_1fr_1fr]`) at `h-[calc(100vh-12rem)]` (accounts for app header + score header), each column wrapped in an `overflow-y-auto` `<Pane>`.

  `useHashTab` uses `useSyncExternalStore` (subscribes to `hashchange`) — the React-blessed pattern for browser-mutable state, replacing an initial draft that used `useState`+`useEffect` (which the new `react-hooks/set-state-in-effect` lint rule flagged as a cascading-render risk). `setTab` calls `history.replaceState` then dispatches a synthetic `hashchange` event so subscribers re-read the snapshot. Slot placeholders ("Plays pane lands in 2.6", etc.) live in `<GameDetail>` until 2.6/2.7/Phase 4 swap them.

- [x] **2.6 `<PlaysPane>` in `src/components/game/plays-pane.tsx`.** Done. `usePlayByPlay(id)` wrapped in `<DataState>` with an 8-row skeleton. Plays sorted by `sortOrder` desc (newest first). - [x] **2.7 `<BoxPane>` in `src/components/game/box-pane.tsx`.** Done. `useBoxscore(id)` wrapped in `<DataState>`. - [x] **2.8 Wire scoreboard tap → set watching + navigate.** Done. `<GameCard>` is now `'use client'` and consumes `useWatching()`. `onClick` on the wrapping `<Link>` calls `setWatching(snapshot)` synchronously before navigation; no `router.push` needed because `<Link>`'s `href` already handles the navigation. Snapshot built from the `ScheduleGame` already in hand (no extra fetch). - [x] **2.9 Wire game-detail mount effect.** Done. Inside `<GameDetail>`: `useEffect` with deps `[id, dataReady, setWatching]` where `dataReady` is `data !== undefined`. The boolean flips false→true once when `useGame` first resolves, and never flips back, so the effect fires on (a) URL id change and (b) initial data arrival — not on every poll. The body builds the snapshot from `data` and calls `setWatching`. Disabled `react-hooks/exhaustive-deps` for this hook because `data` is intentionally NOT in deps (would re-fire on every poll). Live score updates in the pill are covered by 2.10's own `useGame` subscription, not by re-seeding here.

- [x] **2.10 Wire `<NowWatchingPill>` for real.** Done. Outer `NowWatchingPill` early-returns `null` when `watching === null`, then renders an inner `<ActivePill watching={...}>` so the `useGame` subscription can be unconditional inside (rules-of-hooks intact). `useGame(watching.gameId)` shares the same React Query cache key as `/game/[id]` — no duplicate poll; when both pages are loaded the pill updates in lock-step with the score header. Display values prefer `live` (from `useGame`) over the seed snapshot. The intentional snapshot fallback handles the cold-load case (e.g., user opens the app fresh on a non-game page and somehow has a watching value — currently impossible, but defensive). **Deviation:** the spec called for `useGame(id, { enabled })` — useGame's existing signature doesn't take an options bag, and adding one for a single edge case is more work than the early-return-then-mount pattern, which gets the same effect (no fetch when nothing is being watched) without modifying the data layer.

## Working notes (live)

Use this section as a scratchpad while implementing — surprises, decisions, things to revisit. Keep it terse.

- **2026-05-09 — Phase 0.1.** NHL team logos are SVG. Next 16 blocks SVG optimization by default (script-injection risk); enabled with `dangerouslyAllowSVG: true` + the recommended sandboxing CSP. Acceptable risk because the upstream is a single trusted CDN.
- **2026-05-09 — Phase 0.1.** `new URL("https://host/path/**")` shorthand pins `search: ''` (rejects all query strings). NHL serves `…/svg/<TEAM>_light.svg?season=20252026` for some season-variant logos, so I switched to the object form and omitted `search` per the docs ("Omitting the `search` property allows all search parameters"). Acceptable here because the upstream host is trusted; the spec's "minimal surface" instinct loses to the practical "rendered URLs from fixtures must work" requirement.
- **2026-05-09 — Phase 0.1.** Verified the config end-to-end via `curl /_next/image?url=...` against `npm run dev` — `<Image>`-in-throwaway-page is overkill; the optimizer endpoint is the same code path and faster to test. Future image-config changes can use the same shortcut.
- **2026-05-09 — Phase 0.2.** Skipped the `.dark` class hook from the spec (no UI toggle exists yet — would be dead code). Tokens flip via `prefers-color-scheme` only; revisit when a theme toggle ships. Single `--rink-line` token may need to expand to `--rink-red`/`--rink-blue`/`--rink-edge` in Phase 4 once we render the actual rink — punted for now.
- **2026-05-09 — Phase 0.6.** SVG team logos use `unoptimized` on `next/image` because vector optimization is a no-op. Side effect: SVG logos don't actually go through the Next image proxy, so `dangerouslyAllowSVG` from 0.1 is now defense-in-depth only (it'd kick in if some future code rendered an SVG via the proxy without `unoptimized`).
- **2026-05-09 — Phase 0.8.** Plan said `<PlayerChip>` props are `{id, name}` and we'd construct the headshot URL from `id`. NHL headshots are actually keyed by `{season, team, id}`, so id alone isn't enough. Changed props to `{name, headshotUrl?}`; consumers thread the URL from whichever response holds it. Initials fallback when missing.
- **2026-05-09 — Phase 1.6.** Tailwind v4's IDE extension flags `text-[var(--text-muted)]` and suggests the canonical `text-(--text-muted)` form. Both work; `npm run lint` doesn't flag the bracket form. Did the repo-wide sweep in 1.9 once the warnings hit ~20 in one file (`stats-view.tsx`). One `perl -i -pe` against every `.ts`/`.tsx` under `src/`; `globals.css` only had a comment match (updated by hand). One follow-up: the GameStatePill test asserted on `var(--live)` in the rendered className, which became `text-(--live)` after the sweep — fixed in the same commit.
- **2026-05-09 — Phase 1.7.** Next 16 prerender requires any client component using `useSearchParams()` to sit inside a `<Suspense>` boundary. Hit this on the first build of `/standings` ("useSearchParams() should be wrapped in a suspense boundary"). Fix is the standard split: `page.tsx` is a server component that renders `<Suspense fallback={...}><StandingsView /></Suspense>`, and `StandingsView` is the `'use client'` component doing the actual work. Apply the same pattern proactively to any future route reading URL state — `/stats` uses `?kind=`, `/game/[id]` uses URL hash (different — no Suspense needed for `useParams`/hash), Scoreboard date picker in Phase 3 will use `?date=`.
- **2026-05-09 — Phase 2.5.** Eslint's new `react-hooks/set-state-in-effect` rule fires on the standard "subscribe to window event + setState in useEffect" pattern. The right replacement is `useSyncExternalStore` (browser state IS the external store). Used it in `useHashTab` to mirror `useMatchMedia`. `setTab` had to dispatch a synthetic `hashchange` event after `history.replaceState` since replaceState doesn't fire one — `window.dispatchEvent(new HashChangeEvent('hashchange'))` is enough.
- **2026-05-09 — Phase 2.11.** Mocking `window.matchMedia` with `vi.fn()` blew up across tests because `vi.restoreAllMocks()` in `afterEach` resets the mock implementation back to a no-op that returns `undefined`. Subsequent tests would render `useMatchMedia`, get an undefined MediaQueryList, and crash on `.addEventListener`. Switched to plain functions assigned via `Object.defineProperty(window, 'matchMedia', ...)` — they're not vi mocks, so restoreAllMocks leaves them alone. Same pattern in `vitest.setup.ts` as the global default.
- **2026-05-09 — Phase 2.12.** Two Chromium-vs-jsdom landmines in Playwright: (a) `<header>` only gets implicit `role="banner"` when it's a top-level landmark — nested headers don't. jsdom is lenient and matched the integration test's `getByRole("banner")`; Chromium didn't. Fix: add `role="banner"` explicitly. (b) Playwright's `page.goto()` is a full document load — it resets all React state (including the WatchingProvider context). To simulate user navigation that preserves state, click the in-app `<Link>` instead of calling `page.goto`. Both gotchas worth remembering when writing future e2e specs against Next App Router.
- **2026-05-09 — Phase 3.3.** `vi.useFakeTimers()` + `vi.setSystemTime()` to pin the test's "today" was the obvious move but it stalls React Query's internal timers (and thus `waitFor`'s polling), and every test timed out at 5000ms. Mocking the `todayUtcDate` module function via `vi.mock("@/lib/url", ...)` — keeping the rest of the URL helpers real via `vi.importActual` — is the right pattern: it scopes the time-pin to just the value the code reads, leaving timers alone.
